Техническое задание (ТЗ) №5: Менеджер задач для команды разработки
1. Описание проекта
Цель проекта состоит в создании программы для управления задачами в
команде разработки. Проект будет включать минимум 3 класса с шестью полями в каждом,
а также класс для обработки задач с шестью методами, использующими циклы.

2. Классы
2.1. Класс "Task"
Поля:
taskId (): Уникальный идентификатор задачи.
taskTitle (): Название задачи.
taskDescription (): Описание задачи.
taskAssignee (): Имя назначенного исполнителя задачи.
taskStatus (): Статус задачи (новая, в работе, завершена).
taskDeadline (): Крайний срок выполнения задачи.

2.2. Класс "Developer"
Поля:
developerId (): Уникальный идентификатор разработчика.
developerName (): Имя разработчика.
developerEmail (): Электронная почта разработчика.
developerTeam (): Название команды, к которой принадлежит разработчик.
developerTasks (): Список задач, назначенных разработчику.
developerTaskCount (): Количество задач по статусам.

2.3. Класс "TaskManager"
Поля:
taskList (): Список всех задач.
developerList (): Список всех разработчиков.
Методы:
public List<Task> listTasksByAssignee(String assignee): Метод, который
возвращает список задач, назначенных указанному разработчику.
public Map<String, Integer> countTasksByStatus(): Метод, который возвращает
отображение (Map) со статусами задач и количеством задач в каждом статусе.
public void createTask(Task task): Метод, который создает новую задачу и
добавляет ее в список задач.
public void assignTask(Developer developer, Task task): Метод, который назначает
задачу разработчику.
public void updateTaskStatus(Task task, String newStatus): Метод, который обновляет
статус задачи.
public void updateTaskDeadline(Task task, Date newDeadline): Метод, который обновляет
крайний срок выполнения задачи.

3. Обработка данных
Задачи будут создаваться, назначаться, обновляться и фильтроваться с использованием
методов класса "TaskManager". Данные о задачах и разработчиках будут храниться в
соответствующих объектах классов "Task" и "Developer".
